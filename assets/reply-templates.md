# 邮件回复模板 v2.0

三种回复风格，适用于不同场景。使用时将变量替换为实际内容。

---

## 1. 正式版 (Formal)

适用场景：对外客户、上级领导、合作方、正式商务沟通

```
{{greeting_formal}}，

感谢您的来信。

{{body_formal}}

{{action_item_formal}}

如有其他问题，请随时联系。

{{closing_formal}}
{{sender_name}}
{{sender_title}}
```

**变量说明**：

| 变量 | 说明 | 示例 |
|------|------|------|
| {{greeting_formal}} | 正式问候 | "您好 {{recipient_name}}" |
| {{body_formal}} | 正文，完整句子 | "关于您提到的{{topic}}，我们的意见如下：" |
| {{action_item_formal}} | 行动项 | "请于{{deadline}}前确认。" |
| {{closing_formal}} | 正式结尾 | "此致敬礼" 或 "顺祝商祺" |
| {{sender_name}} | 发件人姓名 | "张三" |
| {{sender_title}} | 职位 | "项目经理" |

**示例**：

```
李总，您好，

感谢您的来信。

关于您提到的Q2合作方案，我们已完成内部评估，整体可行。详细方案见附件，请您审阅。

如有其他问题，请随时联系。

此致敬礼
张三
项目经理
```

## 2. 简洁版 (Concise)

适用场景：同事、内部沟通、快速回复、日常事务

```
{{greeting_concise}}，

{{body_concise}}

{{closing_concise}}
```

**变量说明**：

| 变量 | 说明 | 示例 |
|------|------|------|
| {{greeting_concise}} | 简短问候 | "Hi {{recipient_name}}" 或 "{{recipient_name}}，" |
| {{body_concise}} | 直接回答，可用列表 | "确认收到，我会在周五前完成。" |
| {{closing_concise}} | 简单结尾 | "谢谢！" 或 "好的~" |

**示例**：

```
王姐，

确认收到。我会在周五前把报告发给你。

如果需要调整格式，随时说。

谢谢！
```

## 3. 追问版 (Follow-up)

适用场景：信息不足、需要澄清、需要更多细节才能继续

```
{{greeting_followup}}，

感谢您的邮件。关于{{topic}}，我想确认以下几点：

1. {{question_1}}
2. {{question_2}}
3. {{question_3}}

{{context_followup}}

期待您的回复，如有其他信息也可以一并告知。

{{closing_followup}}
```

**变量说明**：

| 变量 | 说明 | 示例 |
|------|------|------|
| {{greeting_followup}} | 问候 | "Hi {{recipient_name}}" |
| {{topic}} | 讨论主题 | "项目进度汇报" |
| {{question_1}} | 需要确认的问题1 | "项目目前的完成比例是多少？" |
| {{question_2}} | 需要确认的问题2 | "预期上线时间是哪天？" |
| {{question_3}} | 需要确认的问题3 | "是否需要和设计团队对齐？" |
| {{context_followup}} | 上下文说明 | "补充这些信息有助于我尽快推进。" |
| {{closing_followup}} | 结尾 | "谢谢！" 或 "期待回复~" |

**示例**：

```
刘哥，

感谢你的邮件。关于新功能开发，我想确认以下几点：

1. 优先级排序是怎样的？P0有哪些功能？
2. 预期上线时间是哪天？
3. 是否需要和设计团队对齐？

补充这些信息有助于我排期和分配资源。

期待你的回复！

小明
```

---

## 通用规则

1. **脱敏**：回复草稿中不包含原文中的敏感信息
2. **确认**：所有回复草稿必须经用户确认后才能发送
3. **定制**：生成时根据邮件内容自动填充变量，不要留空占位符
4. **语气**：根据发件人关系自动选择合适风格（上级用正式，同事用简洁）
5. **安全告警**：安全类邮件的回复通常不需要草稿，直接建议用户处理动作即可
